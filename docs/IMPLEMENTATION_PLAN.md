# 우주 음악 메타버스 구현 계획

## 목표

교사가 Google 로그인으로 수업 세션을 만들고, 학생은 세션코드와 이름으로 입장해 2D 우주 공간에서 행성별 음악을 감상하는 웹앱을 구축합니다. 학생들은 서로의 이름과 위치를 볼 수 있고, 채팅으로 소통할 수 있으며, 교육용 환경에 맞게 기본 비속어 필터를 적용합니다.

## 1차 MVP 범위

1. 교사용 세션 생성 화면
2. 학생용 세션코드/이름 입장 화면
3. 2D 우주 메타버스 공간
4. 수성, 금성, 지구, 화성, 목성, 토성, 천왕성, 해왕성 배치
5. 행성 근처 접근 또는 클릭 시 음악 재생
6. 행성에서 일정 거리 이상 멀어지면 음악 정지
7. 같은 세션에 접속한 학생 이름/위치 표시
8. 세션 채팅과 기본 비속어 필터
9. Firebase Auth/Realtime Database 연결을 위한 환경변수와 보안 규칙 초안

## 기술 선택

- 앱 프레임워크: Next.js + TypeScript
- 2D 메타버스 렌더러: Phaser
- UI: React DOM 오버레이
- 로컬 MVP 상태: Next.js Route Handler + 메모리 저장소
- 운영 확장: Firebase Auth, Realtime Database, RTDB presence/update stream
- 오디오: 브라우저 HTMLAudioElement 기반 MP3 재생

초기 버전은 Firebase 프로젝트 없이도 로컬에서 바로 시연할 수 있게 만듭니다. 이후 Firebase 환경변수를 연결하면 교사 Google 로그인, 학생 익명 인증, 영속 DB, 실시간 위치/채팅 동기화로 확장하는 구조를 유지합니다.

## 아키텍처 원칙

- 게임 규칙과 렌더링을 분리합니다.
- Phaser는 우주 공간, 행성, 아바타, 카메라, 입력 처리에 집중합니다.
- React는 입장 흐름, 채팅, 현재 재생 상태, 교사용 정보 패널을 담당합니다.
- 행성/음원/거리 기준은 안정적인 manifest 데이터로 관리합니다.
- 채팅 필터는 클라이언트와 서버 양쪽에서 적용 가능한 순수 함수로 둡니다.
- 위치 동기화는 1초 단위로 보내고, 화면에서는 보간으로 부드럽게 표시합니다.

## 행성 음원 매핑

- 화성: `6-1-8-01`
- 금성: `6-1-8-02`
- 수성: `6-1-8-03`
- 목성: `6-1-8-04`
- 토성: `6-1-8-05`
- 천왕성: `6-1-8-06`
- 해왕성: `6-1-8-07`
- 지구: `6-1-8-09 푸른 바다 만들기`를 1차 지구 체험곡으로 사용

`6-1-8-08 북극곰아`는 추후 지구 생태/감상 확장 활동에 사용할 수 있도록 보관합니다.

## 화면 구성

### 홈

- 교사 세션 만들기
- 학생 세션 입장
- 앱의 현재 상태와 Firebase 연결 여부 표시

### 교사 화면

- 세션코드 생성
- 학생에게 공유할 코드 표시
- 학생 입장 링크 표시
- 향후 학생 목록/채팅 관리 확장 예정

### 학생 입장

- 세션코드 입력
- 이름 입력
- 입장 전 오디오 재생 동의 역할을 하는 버튼 클릭

### 메타버스

- 중앙 우주 캔버스
- 좌측 또는 상단 현재 감상 행성 패널
- 우측 또는 하단 채팅 패널
- 다른 학생 이름표 표시

## 수용 기준

1. 교사가 6자리 세션코드를 생성할 수 있습니다.
2. 학생이 세션코드와 이름으로 같은 방에 입장할 수 있습니다.
3. 학생 아바타가 키보드 또는 클릭/터치로 이동합니다.
4. 행성 가까이 접근하거나 행성을 클릭하면 해당 음악이 재생됩니다.
5. 다른 행성으로 이동하면 이전 음악은 멈추고 새 음악이 재생됩니다.
6. 행성에서 멀어지면 음악이 정지됩니다.
7. 같은 세션의 다른 학생 이름과 위치가 표시됩니다.
8. 채팅 메시지를 주고받을 수 있습니다.
9. 비속어가 포함된 채팅은 전송되지 않습니다.
10. `npm run build`가 통과해야 합니다.

## 리스크와 대응

- 브라우저 자동재생 제한: 학생 입장 버튼 클릭 후 오디오 객체를 활성화합니다.
- Firebase 미연결 환경: 로컬 메모리 기반 API로 MVP를 먼저 완성합니다.
- 실시간 안정성: 운영 버전은 Firebase Realtime Database 구독으로 세션/채팅/위치를 동기화합니다.
- 음원 권한: 제공된 교육용 음원은 프로젝트 내부 public asset으로 복사하되, 실제 배포 전 사용 권한을 확인해야 합니다.
- 학생 개인정보: 학생은 이름만 입력하고, 운영 버전에서도 이메일 로그인을 요구하지 않습니다.

9. 참가자 위치 동기화와 채팅 UI 구현
