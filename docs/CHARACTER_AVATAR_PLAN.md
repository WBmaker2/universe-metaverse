# 캐릭터 아바타 선택 기능 구현 계획

## 목표

학생이 세션에 입장하기 전에 자신의 2D 캐릭터를 선택하고, 메타버스 공간 안에서 선택한 캐릭터 모습으로 이동하도록 구현합니다. 기존 원형 아바타는 캐릭터 이미지가 없거나 로딩에 실패했을 때의 fallback으로 유지합니다.

## 에셋 선택

- 출처: Kenney Toon Characters
- 공식 페이지: https://www.kenney.nl/assets/toon-characters
- 라이선스: Creative Commons CC0
- 선택 이유:
  - 공식 페이지에서 2D 캐릭터 에셋과 CC0 라이선스를 명시합니다.
  - 교육용 앱에 어울리는 밝고 단순한 스타일입니다.
  - PNG 기반이라 Phaser에서 바로 스프라이트로 사용할 수 있습니다.

## 구현 범위

1. Kenney Toon Characters 에셋 중 5개 캐릭터를 앱용으로 선별합니다.
2. `public/characters`에 캐릭터 이미지를 안정적인 영문 파일명으로 저장합니다.
3. `src/lib/avatars.ts`에 캐릭터 manifest를 추가합니다.
4. 학생 입장 화면에 캐릭터 선택 그리드를 추가합니다.
5. 참가자 데이터 모델에 `avatarId`를 추가합니다.
6. 로컬 세션 API의 join/update/state 응답에 `avatarId`를 포함합니다.
7. Phaser 렌더러에서 원형 아바타 대신 선택한 캐릭터 스프라이트를 표시합니다.
8. 다른 학생도 각자의 선택 캐릭터로 표시합니다.
9. Firebase 참가자 데이터에 `avatarId`를 저장합니다.
10. 스모크 테스트에서 캐릭터 선택 후 캔버스 렌더링을 확인합니다.

## 아키텍처 원칙

- 캐릭터 선택 상태는 participant의 직렬화 가능한 데이터인 `avatarId`로 관리합니다.
- Phaser scene은 `avatarId`를 렌더링에만 사용하고, 참가자 상태의 원천은 React/API 상태로 둡니다.
- 에셋 파일 경로는 코드 곳곳에 직접 쓰지 않고 `AVATARS` manifest를 통해 참조합니다.
- 캐릭터가 없거나 잘못된 `avatarId`가 들어오면 기본 캐릭터로 보정합니다.

## 수용 기준

1. 학생 입장 화면에서 5개 캐릭터 중 하나를 선택할 수 있습니다.
2. 선택하지 않아도 기본 캐릭터로 입장할 수 있습니다.
3. 입장 후 메타버스에서 본인 캐릭터가 원형 대신 캐릭터 이미지로 표시됩니다.
4. 같은 세션의 다른 학생도 각자 선택한 캐릭터 이미지로 표시됩니다.
5. 채팅, 행성 음악 재생, 위치 동기화 기능이 기존처럼 동작합니다.
6. `npm run lint`, `npm run build`, `npm run test:smoke`가 통과합니다.
